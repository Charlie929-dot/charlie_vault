## 📚 内容解读

### 一、结构体基础

1. **为什么需要结构体**  
    结构体可以将不同类型的变量组合成一个整体，便于管理复杂数据（如学生信息包含姓名、学号、成绩等）。
    
2. **结构体定义**
    
    ```
    struct tag {
        member1;
        member2;
    } variable-list;
    ```
    
    - `struct`是关键字
    - `tag`是结构体标签名
    - `member1`, `member2`是成员变量
    - `variable-list`是结构体变量
3. **三种定义方式**
    
    - 先定义类型再定义变量
    - 定义类型同时定义变量
    - 直接定义变量（匿名结构体）

### 二、结构体使用

1. **初始化**  
    可以在定义时直接初始化：
    
    ```
    struct books {
        char title[50];
        char author[50];
    } book = {"C语言", "xingaosheng"};
    ```
    
2. **成员访问**  
    使用`.`运算符访问成员：
    
    ```
    printf("书名: %s", book.title);
    ```
    
3. **结构体数组**  
    数组元素为结构体：
    
    ```
    struct stu {
        char name[20];
        int age;
    } class[5];
    ```
    

### 三、结构体指针（重点）

1. **指向结构体变量**
    
    ```
    struct stu *p = &stu1;
    printf("%s", p->name);  // 使用->访问成员
    ```
    
2. **指向结构体数组**
    
    ```
    struct stu stus[5], *ps;
    for(ps=stus; ps<stus+5; ps++) {
        printf("%s", ps->name);
    }
    ```
    
3. **成员为指针**  
    需要特别注意内存分配：
    
    ```
    struct Student {
        char *Name;
        int number;
    };
    stu.Name = (char*)malloc(sizeof(char));
    ```
    

### 四、typedef与结构体

1. **定义结构体别名**
    
    ```
    typedef struct TreeNode {
        int Element;
        struct TreeNode *Left;
    } Node;  // Node是struct TreeNode的别名
    ```
    
2. **定义结构体指针**
    
    ```
    typedef struct TreeNode *TreePtr;
    TreePtr p;  // 等价于struct TreeNode *p
    ```
    

### 五、实战练习

1. **书籍排序**  
    使用qsort对结构体数组按价格排序
    
2. **成绩排序**  
    多级排序：总分→语文→数学→姓名
    

## 💡 关键点总结

1. 结构体是自定义的复合数据类型，可以包含不同类型的成员
2. 结构体变量使用`.`访问成员，结构体指针使用`->`
3. 结构体指针需要特别注意内存分配问题
4. typedef可以简化结构体类型的使用
5. 结构体在数据结构和复杂数据管理中非常有用
